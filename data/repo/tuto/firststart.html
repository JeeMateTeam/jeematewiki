Les bases Jeedom

Pour une bonne utilisation de JeeMate, il est nécessaire de connaitre les prérequis Jeedom nécessaires à l'autoconfiguration de JeeMate.

L'application Jeemate repose principalement sur deux principes :

  1. La configuration des génériques types de vos équipements
  2. La configuration automatique des équipements issue des plugins gérés nativement par JeeMate

Les génériques types

Les génériques types permettent à différents plugins d'identifier l'équipement et d'adapter sa configuration.

Ex un équipement binaire peut être une lumière, une porte, une fenêtre, un contacteur de chaudière …

Sans le positionnement du type générique, il est impossible pour un Google Home, Alexi, Jeemate de savoir à quoi correspond l'équipement Jeedom.

La bonne nouvelle, c'est que de plus en plus de plugin configurent automatiquement les génériques types, mais d'autres non. Il faudra donc vérifier vos équipements et position le bon type pour les commandes.

Un très bon plugin, Homebridge, à bien documenté cette configuration des génériques types Docu Nebz > Plugin Homebridge (nebzhb.github.io).

Les bases JeeMate

Configuration 

Aller dans le plugin Jeemate “Plugins”⇒"Gestion des plugins"⇒"Communication"⇒"JeeMate"

Configuration des objects

Dans l'onglet “Object” :

  1. cocher les pièces/ objets à remonter dans JeeMate
  2. ouvrir la pièce pour accéder aux équipements

Configuration des équipements

  1. Sélectionner les équipements à remonter dans JeeMate
  2. Ouvrir la page détail. Permets entre autres de configurer les génériques type.
  3. Sauvegarde

Configuration type générique

  1. Choix du générique type
  2. vérification ou modification des liens entre la commande et l'info état de cette dernière
  3. Inverser l'affichage dans JeeMate (un autre tuto présentera toutes les options d'inversion possible)
  4. changer le nom d'affichage dans JeeMate
